# UI位置修复说明

## 🔧 修复内容

### 问题
- 对话框显示在顶部
- 显示不完全
- 位置不稳定

### 解决方案

#### 1. 改用 `fixed` 定位
**修复前**：`position: absolute`
**修复后**：`position: fixed`

**原因**：
- `absolute` 相对于父容器定位，可能受父容器影响
- `fixed` 相对于视口定位，位置更稳定
- 确保UI始终在屏幕可见区域

#### 2. 使用 `vh` 单位
**修复前**：`bottom: 80px`（固定像素）
**修复后**：`bottom: 15vh`（视口高度百分比）

**优势**：
- 适配不同屏幕高度
- 桌面端和移动端统一
- 始终保持在中下部

## ✅ 修复的具体问题

### 1. 对话框显示不完全
- **原因**：`absolute` 定位可能被父容器裁剪
- **解决**：改用 `fixed` + `z-index: 100`

### 2. 位置不稳定
- **原因**：像素值在不同屏幕表现不同
- **解决**：使用 `vh` 单位自适应

### 3. 移动端显示问题
- **原因**：固定像素不适配小屏幕
- **解决**：移动端独立定位（10vh）

### 4. UI层级混乱
- **原因**：未设置 `z-index`
- **解决**：明确设置层级关系

## 🎯 vh单位说明

`vh` = viewport height（视口高度）

- `1vh` = 视口高度的 1%
- `15vh` = 视口高度的 15%

**示例**：
- 屏幕高度 800px → 15vh = 120px
- 屏幕高度 1000px → 15vh = 150px
- 自动适配，无需手动调整

### 移动端（≤ 768px）

## ⚙️ 如何微调位置

### 对话框更高一点
```css
.dialogue-box {
    bottom: 20vh; /* 改大数值 */
}
```

### 对话框更低一点
```css
.dialogue-box {
    bottom: 10vh; /* 改小数值 */
}
```

### 移动端独立调整
```css
@media (max-width: 768px) {
    .dialogue-box {
        bottom: 8vh; /* 移动端专用值 */
    }
}
```
